# AGI MVP — Symbolic Regression via Abstraction & Composition

An experiment in *principled* function discovery. Given a set of (x, y) points
generated by some unknown function, find the **simplest** compositional formula
that explains the data.

This is MVP #1 of a broader AGI research programme built around four pillars:
**feedback loops**, **approximability**, **abstraction & composability**, and
**exploration**. This MVP focuses on the abstraction & composability pillar.

---

## Core idea

Instead of fitting a neural network with millions of opaque weights, we search
over a grammar of human-readable primitives:

```
binary:  +  -  *  /
unary:   sin  cos  square  cube  sqrt  log  exp
```

…and compose them into candidate expressions like `sin(x²) + 2x`. The search
is guided by an **Occam / MDL fitness**:

```
fitness = MSE  +  λ · complexity
```

Adding a node to an expression is only worthwhile if it reduces error by more
than `λ`. This operationalises Occam's Razor: prefer the shortest program
(lowest Kolmogorov complexity) that fits the data.

## Two implementations

### 1. `guess_function.py` — hand-rolled, from scratch

Beam search over expression trees, evaluated in parallel across all CPU cores
using Python `multiprocessing`. Good for understanding how symbolic regression
works under the hood.

```bash
python guess_function.py data.csv
python guess_function.py data.csv --beam 20 --offspring 60 --workers 10 --lam 0.001
```

| Flag | Default | Meaning |
|------|---------|---------|
| `--beam` | 20 | Elite candidates kept each generation |
| `--offspring` | 60 | Mutations per beam member per generation |
| `--workers` | all cores | Parallel worker processes |
| `--gens` | 600 | Max generations |
| `--lam` | 0.001 | Complexity penalty per node |

### 2. `guess_function_pysr.py` — powered by PySR

Wraps [PySR](https://github.com/MilesCranmer/PySR) (Julia backend), which adds
algebraic simplification and returns the full **Pareto front** — one best
expression per complexity level, so you can see the accuracy/simplicity
tradeoff as a table and pick the "knee".

**Setup (one-time):**
```bash
pip install pysr
python -c "import pysr; pysr.install()"   # downloads Julia, ~2 min
```

**Run:**
```bash
python guess_function_pysr.py data.csv
python guess_function_pysr.py data.csv --workers 10 --iters 200 --lam 0.001
```

| Flag | Default | Meaning |
|------|---------|---------|
| `--workers` | all cores | Parallel processes |
| `--iters` | 100 | Search iterations |
| `--maxsize` | 20 | Max expression complexity |
| `--lam` | 0.001 | Complexity penalty (parsimony) |

**Example output:**
```
Complexity  Loss       Equation
─────────────────────────────────────────────
1           3.58       x
3           0.43       x * 2
7    >>>>   1.10e-12   sin(square(x)) + (x + x)   ← Pareto knee
10          1.02e-12   x + (sin(x * x) * 1) + x
```

## Generating test data

```bash
python make_data.py           # writes data.csv (default: sin(x²) + 2x)
python make_data.py out.csv   # custom output path
```

Edit `SECRET_FN` in `make_data.py` to test against any function you like.
